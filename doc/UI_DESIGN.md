## 1. 設計方針

### 1.1 コアコンセプト

**「1レコード = 1行」の原則**
- データベースのレコードと画面の行を1対1対応
- シンプルで予測可能なUI
- データ構造が直感的に理解できる

**「見る → クリック → 操作」の3ステップ**
- 一覧で全体を見る
- 気になる行をクリック
- 詳細画面で操作

### 1.2 ターゲットユーザー

- 家族全員（年齢層：10代〜70代）
- デバイス：スマートフォン、タブレット、PC
- ITリテラシー：初級〜中級

### 1.3 デザイン原則

1. **シンプルさ優先**：必要最小限の情報を表示
2. **タップしやすさ**：ボタンは最小44px × 44px
3. **見やすさ**：適切なコントラスト、大きめの文字
4. **予測可能性**：一貫した操作方法

### I-03: 購入登録（モーダル）

**目的:** 商品購入の登録

**ワイヤーフレーム:**
```
┌────────────────────────────────────┐
│ 購入登録                      [×] │
├────────────────────────────────────┤
│                                    │
│  商品名 *                           │
│  [検索して選択... 🔍]              │
│                                    │
│  数量 *                             │
│  [____] 本                         │
│                                    │
│  保管場所                           │
│  [パントリー ▼]                    │
│  ※ デフォルト値が設定されています   │
│                                    │
│  賞味期限（任意）                   │
│  [____/__/__]                      │
│                                    │
│  購入日（任意）                     │
│  [2025/03/11]                      │
│                                    │
│  ┌──────────────┐                 │
│  │  将来実装予定  │                 │
│  │ [📷 バーコード]│                 │
│  └──────────────┘                 │
│                                    │
│            [登録] [キャンセル]      │
└────────────────────────────────────┘
```

**入力フィールド:**

**商品名（必須）:**
- タイプ：オートコンプリート検索
- プレースホルダー：「商品名を入力...」
- 挙動：
  - 入力開始 → 候補を表示（部分一致）
  - 候補選択 → 保管場所にデフォルト値を自動設定
  - 候補選択 → 単位を自動表示
- データソース：GET /api/products（deleted_at IS NULL）

**実装イメージ:**
```
[しょ_________]
↓ 入力中
┌──────────────┐
│ 醤油 500ml   │
│ 醤油 1L      │
│ 焼酎         │
└──────────────┘
```

**数量（必須）:**
- タイプ：number
- min: 0.1
- step: 0.1
- 単位表示：商品選択後に表示（例：「本」「個」）

**保管場所:**
- タイプ：select
- デフォルト値：商品のdefault_storage_location_id
- 変更可能
- データソース：GET /api/storage-locations

**賞味期限（任意）:**
- タイプ：date
- プレースホルダー：「YYYY/MM/DD」
- 空欄可（NULL）

**購入日（任意）:**
- タイプ：date
- デフォルト値：今日の日付
- 変更可能

**バーコード（将来実装）:**
- グレーアウトボタン
- 「将来実装予定」のラベル

**バリデーション:**
- 商品名：必須
- 数量：必須、0より大きい
- 賞味期限：購入日以降（入力された場合）

**処理フロー:**
1. フォーム入力
2. 登録ボタンクリック
3. バリデーション
4. API呼び出し：POST /api/inventory/purchase
   ```json
   {
     "product_id": 1,
     "quantity": 2,
     "storage_location_id": 1,
     "expiry_date": "2025-03-20",
     "purchased_date": "2025-03-11"
   }
   ```
5. バックエンドでロット検索・更新または新規作成
6. 成功メッセージ表示
7. モーダルを閉じて I-01 を再読み込み

- 保管場所：必須

## 6. UIコンポーネント一覧

### 6.1 ボタン

| タイプ | 色 | 用途 |
|--------|---|------|
| Primary | 緑 #4CAF50 | 主要アクション（登録、追加） |
| Secondary | 青 #2196F3 | 副次アクション（詳細、編集） |
| Danger | 赤 #f44336 | 削除、危険な操作 |
| Default | グレー | キャンセル、閉じる |

**サイズ:**
- 大：48px高さ（モバイル対応）
- 中：40px高さ
- 小：32px高さ

### 6.2 入力フィールド

**テキスト入力:**
- border: 1px solid #ddd
- padding: 10px
- border-radius: 4px
- focus時: border-color #4CAF50

**セレクトボックス:**
- 同上

**日付入力:**
- type="date"
- ブラウザネイティブのピッカー使用

**オートコンプリート:**
- ライブラリ：Vue 3用コンポーネント（例：vue3-simple-typeahead）
- 最低2文字入力で候補表示
- 最大10件表示

### 6.3 テーブル

**スタイル:**
- border-collapse: collapse
- 行ホバー時: 背景色変更
- 行クリック可能: cursor: pointer

**ヘッダー:**
- 背景色: #f8f9fa
- ソート可能カラムにアイコン表示

**ページネーション:**
- 位置：テーブル下部中央
- 表示：「1-20 / 全150件」

### 6.4 モーダル

**サイズ:**
- 小：400px幅
- 中：600px幅
- 大：800px幅

**構造:**
- ヘッダー：タイトル + 閉じるボタン
- ボディ：コンテンツ
- フッター：アクションボタン

**背景オーバーレイ:**
- 透明度50%の黒
- クリックで閉じる

### 6.5 アラート・通知

**成功メッセージ:**
- 背景色：緑
- 位置：画面上部中央
- 表示時間：3秒で自動消去

**エラーメッセージ:**
- 背景色：赤
- 位置：画面上部中央
- 手動で閉じる必要あり
